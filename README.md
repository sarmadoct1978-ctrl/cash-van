# كاش فان — نظام متابعة البروموترز

نظام لإدارة البروموترز، الطلبات، والمخزون — مع تحليل صور وصولات القبض بالذكاء الاصطناعي وتصدير Excel.

## المتطلبات

- Node.js نسخة 18 أو أحدث ([تحميل من هنا](https://nodejs.org))
- مفتاح Anthropic API (احصل عليه من [console.anthropic.com](https://console.anthropic.com/settings/keys))

## خطوات التشغيل

1. **تثبيت الحزم:**
   ```bash
   npm install
   ```

2. **ضبط مفتاح API:**
   انسخ ملف `.env.example` باسم `.env`:
   ```bash
   cp .env.example .env
   ```
   افتح `.env` وضع مفتاحك الحقيقي بدل القيمة الافتراضية:
   ```
   ANTHROPIC_API_KEY=sk-ant-xxxxxxxxxxxxxxxxxxxxxxxx
   ```
   ⚠️ **لا تشارك هذا الملف أو ترفعه على GitHub** — فيه مفتاحك الخاص.

3. **تشغيل السيرفر:**
   ```bash
   npm start
   ```
   راح يشتغل النظام على: `http://localhost:3000`

4. **افتحه من المتصفح** وابدأ الاستخدام.

## كيف يشتغل

- **الواجهة** (`public/index.html`): كل شاشات النظام (البروموترز، الطلبات، المخزون).
- **السيرفر** (`server.js`):
  - `/api/data` — يحفظ ويرجّع بيانات النظام (بدل التخزين المؤقت بالمتصفح، البيانات تنحفظ بملف `data.json` على السيرفر نفسه، يعني تكون متوفرة لجميع الأجهزة اللي تفتح نفس الرابط).
  - `/api/extract-receipt` — يستقبل صورة الوصل من المتصفح، يرسلها إلى Anthropic API **من طرف السيرفر** (مفتاحك ما يوصل أبداً لجهاز المستخدم)، ويرجّع البيانات المستخرجة.
- تفضيل الدور (مدير/بروموتر) يُحفظ محلياً بمتصفح كل جهاز (`localStorage`)، وما يتشارك بين الأجهزة (كل جهاز يختار دوره بنفسه).

## نشر النظام على الإنترنت (اختياري)

أي استضافة تدعم Node.js تناسب (مثل Render، Railway، أو VPS خاص فيك). الخطوات عموماً:
1. ارفع المشروع (بدون `.env` أو `node_modules`)
2. اضبط المتغير البيئي `ANTHROPIC_API_KEY` من إعدادات الاستضافة (وليس بملف نصي)
3. أمر التشغيل: `npm install && npm start`

## ملاحظة عن `data.json`

هذا الملف يخزّن كل بيانات النظام (البروموترز، الطلبات، المخزون) بصيغة JSON بسيطة. يكفي لعدد صغير-متوسط من البروموترز والطلبات. إذا كبر النظام مستقبلاً وصار عندك مئات الطلبات يومياً، الأفضل الانتقال لقاعدة بيانات حقيقية (مثل SQLite أو PostgreSQL) — أقدر أساعدك بهذا التحويل وقتها.
